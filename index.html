ASCIIROG

ASCII 로그라이크

@, #, . 문자로 그려지는 턴제 로그라이크. 고전 로그라이크 규칙 위에 물리·열역학 시뮬레이션 레이어를 얹었습니다.

핵심 문제

환경 상호작용을 정해진 연출로 재생하지 않고, 모든 타일에 물질·상태·온도를 부여해 규칙끼리 상호작용하도록 만들었습니다. 불은 가연성 타일을 따라 번지고, 기름에 옮겨붙으면 폭발하며, 물은 얼거나 끓어 증기가 됩니다. 번개는 BFS로 전도성·젖은 타일을 따라 전파됩니다. 결과적으로 미리 설계하지 않은 조합도 성립합니다.

  • Strategy 패턴 맵 생성 — BSP 던전 / 셀룰러 오토마타 동굴 / 펄린 노이즈 지형을 같은 인터페이스로 교체합니다.
  • 타일별 조명 — 거리 감쇠 + 브레젠험 직선으로 벽 차폐를 계산해 문자 단위로 밝기를 입힙니다.
  • 성능 — 5000 슬롯 파티클 풀, 적 탐색용 SpatialHashGrid, Pub-Sub EventManager.
  • 데이터 드리븐 — 바이옴·물질·파티클을 CSV로 정의하고 에디터 임포터가 ScriptableObject로 변환합니다.

RTXX

실시간 전략 · Total War 계열

부대 단위로 조작하는 RTS. 우클릭으로 선을 그어 부대의 정면과 폭을 지정하면 병사들이 해당 대형으로 정렬합니다. 2D 픽셀 스프라이트 병사가 3D 지형 위에서 전투합니다.

핵심 문제

병사 200명을 대형 슬롯 200개에 배정하는 문제입니다. 가까운 슬롯부터 그리디하게 배정하면 이동 경로가 교차해 대형이 엉킵니다. 이를 이분 그래프 최적 할당으로 정의하고 헝가리안 알고리즘을 직접 구현해 총 이동거리가 최소인 배치를 계산했습니다. 같은 알고리즘을 부대 ↔ 진형 선분 배정에도 재사용합니다.

  • DOTS 병렬화 — 부대/유닛/전투/애니메이션/스탯/대형 갱신을 전부 [BurstCompile] IJobParallelFor로 처리하고, 지형 접지 판정은 RaycastCommand로 배치 처리합니다.
  • 중앙 집중 업데이트 루프 — 모든 MonoBehaviour를 비활성화하고 Controller가 _Update()를 직접 호출합니다. Unity의 Update() 호출 오버헤드를 줄이기 위한 구조입니다.
  • 각도 기반 전투 — 정면 피격에는 방어력·방패가 100%, 측후면에는 50%만 적용됩니다. 측면 공격이 전술적 의미를 갖습니다.
  • 3초간 교전이 없으면 자동 재정렬로 무너진 대형을 복구합니다.
Unity 2022.3URP DOTS / BurstJob System 헝가리안 알고리즘

CarDrive

운전 서바이벌 호러

밤길을 달리는 1인칭 운전 게임. 주행 중 귀신이 차체에 달라붙어 체력을 깎고, 플레이어는 앙크를 충전해 퇴치합니다. 운전과 전투를 동시에 수행해야 합니다.

핵심 문제

운전 자체에 부하가 없으면 전투와 병행할 이유가 사라집니다. 차량을 WheelCollider 기반 물리로 구현하고 CarData에 토크 곡선·기어비·연료 소모를 데이터로 분리했습니다. 자동 변속과 시동(E키)을 넣어, 귀신이 붙었을 때 정차 후 처리할지 주행하며 버틸지가 선택지가 되게 했습니다. 귀신은 시동이 걸려 있을 때만 스폰됩니다.

  • 무한 도로 스트리밍 — RoadManager가 도로 세그먼트를 앞쪽에 생성하고 뒤쪽을 회수해, 끝없이 이어지는 밤길을 만듭니다.
  • 퇴치 판정 — 마우스 좌클릭으로 앙크를 충전하고 SphereCast로 달라붙은 귀신을 맞힙니다.
  • 레트로 렌더링 — 커스텀 URP 렌더러 피처로 픽셀화, 팔레트 색상 양자화, 디더링을 적용했습니다.

Sandland

드론 지휘

1인칭으로 관제실을 이동하다 CRT 모니터 앞에 서면 제어 모드로 전환되고, 모니터 화면을 통해 정찰 드론 편대를 지휘합니다. 화면은 RenderTexture에 픽셀화·CRT 셰이더를 적용해 출력합니다.

핵심 문제

드론 편대가 대형(Line / Circle / V / Diamond)을 유지하며 NavMesh 위를 비행해야 했습니다. 어떤 드론을 어떤 대형 슬롯에 배정할지가 문제였고, RTXX와 동일하게 헝가리안 알고리즘으로 최적 배정을 계산했습니다. 카메라·수송·공격·호위·방어 드론이 역할별 상태 머신(약 1,400줄)에 따라 동작합니다.

  • 구체 스캔 시스템 — 스캔 범위가 구체로 퍼져나가며 아이템과 적을 태깅합니다.
  • 무기 / 투사체 — 머신건·미사일·레이저, 탄도 및 유도 투사체, 오브젝트 풀링.
  • 관제 UI — 레이더, 고도계, 나침반, 드론 상태, 아이템 관리 패널을 모니터 안에 구현.

0MOJEON

부머 슈터

3D 공간에 2D 빌보드 스프라이트 적이 배치되는 레트로 FPS. 픽셀화, 색상 팔레트 양자화 렌더러 피처로 90년대 FPS의 화면을 재현했습니다.

핵심 문제

부머 슈터는 이동이 핵심인 장르라 이동 처리에 비중을 뒀습니다. Rigidbody 가속/마찰 기반 이동에 공중 대시, 벽 슬라이딩, 계단 자동 오르기를 추가해 속도가 끊기지 않도록 만들었습니다.

  • 데이터 드리븐 무기 — WeaponData를 ScriptableObject로 두고 투사체 / 히트스캔 / 근접 / 차지 공격을 조합합니다.
  • 적 AI — Idle / Chase / Attack / Reposition FSM에 4가지 행동 타입, NavMesh 이동.
  • 아레나 구조 — 게이트·열쇠, 픽업, 함정, 오브젝트 풀링, 스탯 수정자 시스템.

MX4

청소 시뮬레이터 하드웨어 연동

마우스를 드래그해 3D 오브젝트 표면의 얼룩을 물로 닦아내는 게임. Logitech Action Ring으로 도구를 전환하고, MX Master 4 햅틱으로 피드백을 전달합니다.

핵심 문제

얼룩의 제거 진행도를 매 프레임 계산해야 했습니다. 얼룩을 텍스처가 아니라 UV 공간의 RenderTexture 마스크로 두고 HLSL 브러시 셰이더가 직접 그려 넣습니다. 진행도는 마스크를 CPU로 읽는 대신 밉맵을 생성해 최하위 1×1 밉이 전체 평균이 되는 성질로 구했습니다. 데스크톱에서는 AsyncGPUReadback으로 비동기 처리하고, 이를 지원하지 않는 WebGL에서는 셰이더로 1×1까지 blit한 뒤 동기 리드백하는 폴백 경로를 뒀습니다.

  • 노즐 반경에서 수압 파생 — 수압을 별도로 조절하지 않고 pressure = (기준반경 / 노즐반경)^감쇠로 계산합니다. 좁은 노즐은 강하고 좁게, 넓은 노즐은 약하고 넓게 작용해 범위와 세기의 트레이드오프가 생깁니다.
  • 연속 스트로크 — 프레임 사이 마우스 이동 거리를 보간해 스탬프를 여러 번 찍습니다.
  • 게임 ↔ 플러그인 연동 — 게임이 loopback TCP 서버(52111)로 상태·이벤트를 전송하고, 별도 .NET 8 Loupedeck 플러그인이 이를 받아 링 디스플레이와 햅틱을 구동합니다.

SCPPJ

액션 어드벤처 팀 프로젝트

전투 · 잠입 · 단서 추론을 결합한 3D 액션 어드벤처. 스크립트 약 338개 규모이며, 참여한 프로젝트 중 아키텍처 설계 비중이 가장 큰 작업입니다.

핵심 문제

팀 규모와 기능 수가 늘면서 시스템 간 결합이 빠르게 증가했습니다. VContainer DI로 스코프(Root / Gameplay / Player / Combat / UI / Puzzle)를 분리해 조립하고, 시스템 간에는 직접 참조 대신 struct 이벤트 버스로 통신하게 했습니다. 갱신은 개별 Update() 대신 중앙 틱 매니저가 ITickable을 호출합니다.

  • 마인드 스페이스 — 수집한 단서를 연결해 레시피와 매칭하면 추론이 성립하고 진행도·카드가 해금됩니다.
  • 스텔스 — 시야 감지와 소음 전파 기반 청각 감지의 2채널 구조. 몬스터 AI는 Idle/Patrol/Chase/Attack/Investigate/Distracted FSM입니다.
  • 카드 시스템 — Strategy 패턴으로 즉발/투척/설치를 분리하고, 손패 UI는 MVVM으로 구성했습니다.
  • 물리 인벤토리 — 가방 안에서 아이템이 물리적으로 움직이며, 마우스로 집어 옮길 수 있습니다.
